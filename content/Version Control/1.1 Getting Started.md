### Versionamento

**controle de versão** é um sistema que registra alterações em um arquivo ou conjunto de arquivos, com o intuito de guardar suas versões ao longo do tempo.

*VCS*(Sistema de controle de versão) é o sistema que é responsável por fazer esse controle.

utilizar ferramentas *VCS* permite reverter para versões anteriores, saber quem fez alterações que causaram problemas, quem introduziu o problema e muito mais.
VCS permitem recuperações caso de perda dos arquivos.

Existem 2 tipos de controle de versão, **DVCS** e **CVCSs**, sistemas distribuidos de controle de versão e sistemas centralizados de controle de versão.

#### CVCSs

se caracterizam por terem um servidor principal onde cada desenvolver trabalha diretamente nele.

esse formato não é recomendado pois caso o sistema fique fora do ar por 1 hora, será 1 hora sem poder realizar alterações no servidor.

também é pessímo pois caso ocorra corrupção dos arquivos e o backup não servir, todas as alterações do projeto seram perdidas.
#### DVCS
se caracterizam por terem um servidor principal com estado atual do projeto, e cada desenvolvedor possui uma copia exata dos arquivos dos servidores.

Assim, se qualquer servidor morrer, e esses sistemas estiverem colaborando por meio dele, qualquer um dos repositórios de clientes podem ser copiado de volta para o servidor para restaurá-lo. Cada clone é de fato um backup completo de todos os dados.
